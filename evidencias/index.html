Este proyecto integra la configuración de entornos virtuales, contenedores desacoplados y orquestación con Kubernetes para desplegar una infraestructura robusta y escalable.

Distribuciones: Ubuntu 24.04 (Gráfica) / Debian 13.5 (Consola)

Componente 1: Virtualización

Instalación y configuración de dos nodos independientes con aislamiento de red y acceso seguro vía SSH.

  • Comandos clave: ip a, lsblk, ssh usuario@ip

🐳 Componente 2: Contenedores Docker

Orquestación de microservicios desacoplados (Nginx + Python) usando Docker Compose y volúmenes persistentes.

  • Frontend: Nginx en puerto 8080 (mapeado al 80).
  • Backend: Python API en puerto 5000.

☸️ Componente 3: Kubernetes

Despliegue escalable en Minikube con servicios NodePort y alta disponibilidad.

  • Servicio: NodePort expuesto en puerto 30080.
  • Escalabilidad: 3 réplicas activas verificadas.

Conclusiones

Aprendizaje: La orquestación con Kubernetes permite una gestión de fallos automática inalcanzable con contenedores individuales.

Dificultad: El mapeo de puertos entre el host físico y el clúster de Minikube requirió ajustes en las reglas de firewall.

Recomendación: Implementar un pipeline de CI/CD para automatizar las actualizaciones del frontend.
